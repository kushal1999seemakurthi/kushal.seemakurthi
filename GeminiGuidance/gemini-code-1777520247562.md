# 🌌 Aegis Immersive: Visual & Interactive UI Specifications

> **Agent Directive:** This document dictates the precise user experience, visual layout, and interaction logic for the 3D celestial portfolio. Implement these behaviors exactly as described, focusing on smooth camera interpolation and dynamic object rendering.

## 1. The Initial Landing State (The Intro Island)
When the user first loads the page, the environment must immediately establish a sense of depth and scale.

* **The Environment:** The scene is set in deep space with surrounding, volumetric nebula clouds. 
* **The Current Location:** The camera is positioned in the middle of a floating island (The "Intro" Island).
* **The Horizon:** The *next* island (representing "Skills") is visible floating far off in the distance. 
* **No Initial Path:** There are **no stairs or bridges visible** upon landing. The islands are visually disconnected.
* **Ethereal UI:** General introductory information (who I am, what I do) is not placed on a harsh HTML backdrop. Instead, this text is rendered as floating elements directly within the free space of the nebula clouds to the left or right of the island.

## 2. Interactive 3D Elements (In-World Links)
Instead of standard HTML buttons, the island contains natural, thematic 3D objects that serve as navigation points.

* **Object Design:** The Intro island features natural entities, such as aesthetically pleasing rocks or crystals, scattered around the terrain.
* **Engravings & UI:** These natural objects feature engraved symbols. For example, a prominent, beautiful rock will have the LinkedIn logo engraved into its surface.
* **Interaction Logic (Raycasting):** These 3D objects must be interactive. When the user hovers over the LinkedIn rock, it should provide a visual cue (e.g., a subtle glow or scale increase). Clicking the rock acts as an anchor tag, routing the user externally to the LinkedIn profile.

## 3. The Scroll Transition & Dynamic Path Generation
Scrolling does not move a standard webpage down; it moves the camera forward through the 3D space and triggers world-building animations.

* **Camera Movement:** As the user begins to scroll, the camera smoothly moves from the center of the Intro island towards the leading edge facing the Skills island.
* **Dynamic Stair Formation:** As the camera approaches the edge and continues forward, a path begins to form *dynamically*. 
* **Animation Specs:** Individual stair steps do not just fade in; they physically fly in from the left and right out of the surrounding nebula clouds, assembling themselves step-by-step directly in front of the camera to form the bridge to the next island.
* **Continuous Flow:** This dynamic generation must feel synchronized with the user's scroll speed (using a library like GSAP ScrollTrigger), meaning if they stop scrolling, the stairs stop forming.
