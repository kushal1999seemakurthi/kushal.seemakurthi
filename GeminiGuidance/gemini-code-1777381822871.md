# 🧊 Aegis Immersive: The Zero-Server 3D Web Blueprint

Welcome to the **Aegis Immersive Architecture**. This repository serves as a boilerplate and documentation guide for building, testing, and deploying Awwwards-tier, high-performance WebGL experiences. 

By leveraging client-side GPU rendering, autonomous QA loops, and edge CDNs, this architecture delivers cinematic 3D web experiences without the need for expensive backend servers.

---

## 🏗️ 1. The Core Stack

We treat the browser as a real-time rendering engine, bypassing traditional DOM manipulation where possible.

* **3D Engine:** `Three.js` (via `three-mesh-bvh` for optimized raycasting).
* **Framework:** `Svelte` (Zero virtual-DOM overhead, ideal for tight frame-by-frame WebGL sync).
* **Motion & State:** `GSAP` + `ScrollTrigger` + `Lenis` (Smooth scroll interpolation).
* **Build Tool:** `Vite` (Crucial for HMR during GLSL shader development).

## ⚡ 2. Edge-Ready Asset Optimization 

Because this application is hosted purely on a CDN (Cloudflare Pages), **asset size is the primary bottleneck**. The client's GPU does the heavy lifting, but we must ensure the initial payload is downloaded instantly. 

**Strict Asset Rules:**
1. **Draco Compression:** All `.glb` / `.gltf` 3D models MUST be compressed using the Google Draco algorithm. Do not load raw, uncompressed meshes.
2. **Texture Baking:** Real-time lighting calculation kills frame rates on mobile devices. Bake shadows, ambient occlusion, and lighting directly into your textures using Blender before exporting. Keep Three.js lights to an absolute minimum (0 to 1).
3. **KTX2 / WebP Textures:** Never use standard `.png` or `.jpg` files for 3D textures. Convert all textures to `.ktx2` format, which can be sent directly to the GPU without unzipping, saving massive amounts of VRAM.
4. **Procedural Generation:** Use code to generate repeating patterns, particle systems, or noise via GLSL Shaders instead of loading large image files.

## 🤖 3. The Auto-Healing QA Loop (Aegis)

WebGL bugs are notoriously difficult to track. This repository utilizes an LLM-powered Auto-Healing CI/CD pipeline to catch visual regressions and patch them autonomously.

### How it Works:
1. **The Trap:** On every push, headless **Playwright** runs functional and visual regression tests (verifying canvas context, shader compilation, and scroll-scrubbing bounds).
2. **Context Aggregation:** If a test fails, a local Node script bundles the stack trace, the browser console logs, the Playwright visual diff, and the recent Git commits.
3. **Agentic Patching:** This context is piped into an AI CLI Agent (e.g., Aider via the Gemini API). The agent attempts to fix the Three.js or state management bug.
4. **Validation:** The fix is applied to a detached branch, tests run again, and if successful, an `Auto-Fix PR` is generated for human review.

## 🚀 4. Deployment (Cloudflare Pages)

This site requires **zero backend servers**. It operates entirely via static file delivery and client-side rendering.

**Infrastructure Cost:** ~$10/year (Domain registration only).

**Deployment Steps:**
1. Register your domain via **Cloudflare Registrar**.
2. Connect this GitHub repository to **Cloudflare Pages**.
3. Set the build command: `npm run build`
4. Set the output directory: `dist`
5. **Result:** Every git push will trigger a free build. Cloudflare will automatically distribute your Draco-compressed models and KTX2 textures to edge servers globally, ensuring millisecond latency worldwide.

## 📜 5. Developer Guardrails & Best Practices

When contributing to or extending this codebase, adhere to these rules:

* **Scroll as Progress:** Do not use `IntersectionObserver` for continuous animations. Map the window's scroll position to a `0.0` to `1.0` progress float, and use that single variable to scrub a master GSAP timeline.
* **Instanced Rendering:** If you need 1,000 floating particles or debris objects, you MUST use `InstancedMesh`. Never create 1,000 separate `Mesh` objects.
* **Memory Leaks:** Three.js does not automatically garbage collect. When unmounting a Svelte component, you must explicitly call `.dispose()` on all geometries, materials, and textures.
* **No DOM Reflows:** Keep the UI layer (`<div>`, `<p>`) completely separate from the Canvas. Animate the Canvas layer using WebGL, and animate the UI layer using only CSS `transform` and `opacity` to prevent layout recalculations.
